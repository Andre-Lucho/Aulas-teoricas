## Void

### No JavaScript, uma função sem return irá retornar undefined.<br> No TypeScript o retorno é definido como void.<br> Isso evita usos errados como checagens booleanas de métodos que não possuem retorno.

```
function pintarTela(cor: string) {
  document.body.style.background = cor;
}

pintarTela('black');

// Erro, void não pode ser verificado
if (pintarTela('black')) {
}

const btn = document.querySelector('button');

// Erro, void não pode ser verificado
if (btn && btn.click()) {
}

```

## Never

### O never é utilizado em casos onde a função gera um erro ou termina a aplicação.

```
1.)

function abortar(mensagem: string): {
  throw new Error(mensagem);
}

abortar('Um erro ocorreu'); // lança um erro e para a execução
console.log('Tente novamente'); // não é executado, pois o bloco de código trava após a execução de abortar

```

```
2.)

function abortar(mensagem: string): never {
  throw new Error(mensagem);
}

abortar('Um erro ocorreu'); // lança um erro e para a execução
console.log('Tente novamente'); // NUNCA será executado - "código inacessívelo detectado" == prórpio VSCode(lint) apaga o código.

```

## Overload

### Existem funções que retornam diferentes dados dependendo do argumento. Podemos declarar a interface dessas funções utilizando function overload.<br> Basta declarar a interface antes da definição da mesma, utilizando o mesmo nome.<br> O Overload deve ser compatível com a função original.

```
// Exemplo 1
function normalizar(valor: string[]): string[];
function normalizar(valor: string): string; *
function normalizar(valor: string | string[]): string | string[] {
  if (typeof valor === "string") {
    return valor.trim().toLowerCase();
  } else {
    return valor.map((item) => item.trim().toLowerCase());
  }
}


normalizar(" Produto ");
normalizar(["Banana ", " UVA"]);

// Exemplo 2

function $(seletor: number): number; // ERRO **
function $(seletor: "video"): HTMLVideoElement | null;
function $(seletor: "div"): HTMLDivElement | null;
function $(seletor: "a"): HTMLAnchorElement | null;
function $(seletor: string): Element | null;
function $(seletor: string): Element | null {
  return document.querySelector(seletor);
}

$("a")?.href;
$("video")?.volume;
$(".teste")?.innerHTML;


```

## Importante

### 1) \* Devo sempre declarar o overload 'original' sempre antes da função original: <br>A função original recebia uma string ---> devo declarar pelo menos um overload recebendo e retornando 'string' e depois as demais;

### 2) Todos os overloads devem estar acima e imediatamente antes da função original.

### 3) \*\* Erro acima --> a função original não recebe number, por isso o overload não pode receber number. <br>Ela recebe um valor do tipo 'string'; observar que todos os demais overloads tem interfaces que são heraças uns dos outros, além de serem strings!!!

## Exercicio

// Crie uma função que arredonda um valor passado para cima.
// A função pode receber string ou number.
// A função deve retornar o mesmo tipo que ela receber.

## Solução

```
function arredondar(valor: string): string;
function arredondar(valor: number): number;
function arredondar(valor: string | number): string | number {
  if (typeof valor === 'number') {
    return Math.ceil(valor);
  } else {
    return Math.ceil(Number(valor)).toString();
  }
}

console.log(arredondar(200.32));
console.log(arredondar('200.32'));

```
