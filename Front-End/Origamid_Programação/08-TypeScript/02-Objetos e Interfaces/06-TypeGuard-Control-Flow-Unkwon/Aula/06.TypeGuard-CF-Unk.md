# Type Guard e Control Flow

## Guard, Safety e Narrowing

### O Type Guard (defesa) garante a Type Safety (segurança) do dado dentro do bloco condicional. Esse processo é chamado de Type Narrowing (estreitamento). <br>O TypeScript faz Control Flow (controle de fluxo) para entender qual o dado dentro da condicional.

```
/function typeGuard(value: any) {
  if (typeof value === 'string') {
    return value.toLowerCase();
  }
  if (typeof value === 'number') {
    return value.toFixed();
  }
  if (value instanceof HTMLElement) {
    return value.innerText;
  }
}

typeGuard('Origamid');
typeGuard(200);
typeGuard(document.body);

```

## Unknown

### Indica que não sabemos o tipo de dados que pode ser passado. Diferente do any, o 'unknown' NÃO ASSUME nenhum tipo de dado e teremos, obrigatóriamente, que usar uma Type Guard (e garantir uma Type Safety) para utilizar métodos dentro da função. <br> \*OBS. 'any' assume que o parâmetro pode ser de qualqual tipo de dado

```
function typeGuard(value: unknown) {
  if (typeof value === 'string') {
    return value.toLowerCase();
  }
  if (typeof value === 'number') {
    return value.toFixed();
  }
  if (value instanceof HTMLElement) {
    return value.innerText;
  }
}

typeGuard('Origamid');
typeGuard(200);
typeGuard(document.body);


```
