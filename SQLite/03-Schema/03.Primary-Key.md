## Primary Key

### Primary Key (pk)

Define uma coluna como chave primária, que deve ser **única e não nula**.
Palavra reservada: **PRIMARY KEY**

```sql
CREATE TABLE "cursos" (
  "id" INTEGER PRIMARY KEY,
  "nome" TEXT,
  "preco" INTEGER
) STRICT;

INSERT INTO
  "cursos" ("nome", "preco")
VALUES
  ('HTML', 1000);

```

### OBS:

---

#### 1. NÃO POSSO inserir o mesmo dado em campos com PRIMARY KEY == erro

#### 2. Se eu fizer um novo insert sem nenhum dados --> como ele é único <br>== Auto-incrementa o campo id com um novo número +1 em relação ao anterior!!

#### 3. O campo com PRIMARY KEY será sempre NOT NULL

## RowId

(_Importante na indexação de dados do SQlite_)

<br>

Toda tabela tem uma coluna oculta chamada **rowid**, que é um identificador único para cada linha.

Se você não definir uma chave primária, o SQLite cria automaticamente uma coluna **rowid**.

Se você definir uma coluna como **INTEGER PRIMARY KEY**, ela se torna um alias para a coluna **rowid**, isso não acontece caso use **INT**.

```sql
SELECT "rowid", * FROM "cursos";
```
