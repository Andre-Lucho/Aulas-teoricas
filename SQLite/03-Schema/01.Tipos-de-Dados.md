## INTEGER e TEXT

Esses são os dois tipos de dados mais comuns no SQLite.

### INTEGER

Armazena números inteiros (boolean (0,1), timestamp)

### TEXT

Armazena textos (strings, datas, json)

## Dados no SQLite

Existem 3 momentos importantes em que o SQLite lida com tipos de dados:

### Definição de Tabelas

Definir o tipo de dado da coluna.

### Inserção de Dados

Converte o dado para o tipo de acordo com a afinidade da coluna.

### Consulta de Dados

Retorna o valor armazenado.

## Tipos de Dados

O SQLite possui os seguintes tipos de dados:

### TEXT

- Texto

### INTEGER

- Números inteiros

### REAL

- Números flutuantes

### BLOB

- Dados binários

### NULL

- Valor nulo

## Afinidade de Tipo

SQLite não impõe tipos de dados estritos, mas sim uma afinidade de tipo.  
Isso significa que você pode armazenar qualquer tipo de dado em qualquer coluna, mas o SQLite tentará converter os dados para o tipo mais apropriado, de acordo com a afinidade da coluna.

### São 5 tipos de afinidade:

### TEXT

-Texto

### INTEGER

-Números inteiros

### REAL

-Números flutuantes

### BLOB

-Dados binários

### NUMERIC

-Números inteiros ou flutuantes

https://www.sqlite.org/datatype3.html

```sql
CREATE TABLE "produtos" (
  "id" INT,
  "descontinuado" TINYINT,
  "nome" VARCHAR(100),
  "preco" DECIMAL(10, 2),
  "descricao" TEXT,
  "data_criacao" DATETIME
);

INSERT INTO
  "produtos" ("id", "descontinuado", "nome", "preco", "descricao", "data_criacao")
VALUES
  (1, 0, 'Notebook', 200.50, null, '24-10-2049');

SELECT
  "id", typeof("descontinuado"), typeof("preco"), typeof("data_criacao"), typeof("descricao")
FROM
  "produtos";

```

## Conversão de Tipos

O SQLite converte automaticamente os tipos de dados quando necessário. Por exemplo, se você inserir um número em uma coluna TEXT, o SQLite armazenará esse número como texto.

```sql
INSERT INTO
  "produtos" ("id", "descontinuado", "nome", "preco", "descricao", "data_criacao")
VALUES
  ('1', '0', 999, '200.50', 100, 2049);

```
