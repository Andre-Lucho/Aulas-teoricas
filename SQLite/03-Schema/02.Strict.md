# Strict

No modo strict, o SQLite impõe regras mais rígidas sobre os tipos de dados.
Palavra reservada: STRICT

- OBS: posso ter tabelas no modo strict e outras no modo padrão em um mesmo arquivo .sqlite

## CREATE TABLE

Apenas alguns tipos de dados são permitidos.

## INSERT

Os dados devem corresponder ao tipo definido na tabela.

```sql
CREATE TABLE "cursos" (
  "id" INTEGER,
  "nome" TEXT,
  "preco" INTEGER
) STRICT;

INSERT INTO
  "cursos" ("id", "nome", "preco")
VALUES
  ('a', 'HTML', 1000); -- ERRO, 'a' não é um INTEGER

INSERT INTO
  "cursos" ("id", "nome", "preco")
VALUES
  ('1', 'HTML', 1000); -- Funciona, transforma '1' em 1
```

## Tipos de Dados (STRICT)

SQLite no modo strict aceita a criação de tabelas com os seguintes tipos de dados:

1.  INT\*

    Números inteiros

        OBS: TIPO 'INT' NÃO SE ASSOCIA A 'rowid' - olhar próxima aula

2.  INTEGER

    Números inteiros

3.  TEXT

    Texto

4.  REAL

    Números flutuantes

        OBS: TIpo Real NÃO SÃO PRECISOS - dar preferência ao INTEGER e tranformar o dado no backend

5.  BLOB

    Dados binários

6.  ANY

    Qualquer tipo de dado

```sql
CREATE TABLE "cursos" (
  "id" INTEGER,
  "nome" TEXT,
  "preco" INTEGER
) STRICT;
```
