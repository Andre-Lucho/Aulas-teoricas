## Strict

No modo strict, o SQLite impõe regras mais rígidas sobre os tipos de dados.

### CREATE TABLE

Apenas alguns tipos de dados são permitidos.

### INSERT

Os dados devem corresponder ao tipo definido na tabela.

```sql
CREATE TABLE "cursos" (
  "id" INTEGER,
  "nome" TEXT,
  "preco" INTEGER
) STRICT;

INSERT INTO
  "cursos" ("id", "nome", "preco")
VALUES
  ('a', 'HTML', 1000); -- ERRO, 'a' não é um INTEGER

INSERT INTO
  "cursos" ("id", "nome", "preco")
VALUES
  ('1', 'HTML', 1000); -- Funciona, transforma '1' em 1

```

## Tipos de Dados (STRICT)

SQLite no modo strict aceita a criação de tabelas com os seguintes tipos de dados:

### INT

\*: Números inteiros

### INTEGER

Números inteiros

### TEXT

Texto

### REAL

Números flutuantes

### BLOB

Dados binários

### ANY

Qualquer tipo de dado

```sql
CREATE TABLE "cursos" (
  "id" INTEGER,
  "nome" TEXT,
  "preco" INTEGER
) STRICT;

```
